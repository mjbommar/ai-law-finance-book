\section{Grounding and Retrieval Basics (RAG 101)}
\label{sec:llmC-rag}

% Outline (comments):
% - Why grounding: verification, recency, jurisdiction, provenance
% - Chunking strategies (by structure vs. fixed size); overlap; identifiers
% - Embeddings/indexing; filters (jurisdiction, date, client)
% - Freshness/versioning; re-index triggers
% - Citation fidelity: quotes, page/para IDs, timestamps, stable URLs/DOIs

Embeddings provide the similarity signal that powers retrieval; see the primer in \Cref{sec:llmA-embeddings}. All retrieval outputs should carry the canonical evidence record defined in \Cref{sec:llmC-evidence}.

\begin{definitionbox}[title={Grounding Goals}]
\begin{itemize}
  \item Tie every claim to sources with quotes and identifiers.
  \item Filter by jurisdiction, date ranges, and client permissions.
  \item Refresh indexes on document updates; record versions.
\end{itemize}
\end{definitionbox}
