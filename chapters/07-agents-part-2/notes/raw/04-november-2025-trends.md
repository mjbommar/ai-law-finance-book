## Executive Summary

**Major AI Agent Developments (Late 2024 - November 2025)**:

1. **Protocol Standardization**: Two competing standards emerged—Anthropic's Model Context Protocol (MCP, November 2024) and Google's Agent-to-Agent (A2A, April 2025)—with MCP achieving rapid adoption from OpenAI, Google DeepMind, and Microsoft

2. **Computer Use Breakthrough**: Anthropic's Claude models achieved 61.4% on OSWorld benchmark (September 2025), demonstrating practical autonomous desktop control capabilities

3. **Enterprise Platforms Mature**: Amazon Bedrock AgentCore (GA October 2025), Microsoft Agent 365 (November 2025), and major framework releases (LangChain/LangGraph 1.0, October 2025) signal production readiness

4. **Legal AI Goes Agentic**: Harvey AI ($5B valuation, June 2025) and Thomson Reuters CoCounsel (August 2025) launched multi-step reasoning agents for complex legal workflows

5. **Security and Governance Emerge**: EU AI Act agent requirements clarified, AWS released Agentic AI Security Scoping Matrix, OpenAI launched Aardvark security agent (October 2025)

## Protocol Developments

### MCP (Model Context Protocol)

**Adoption Timeline**:
- Nov 2024: Anthropic launches MCP
- Mar 2025: OpenAI adopts (ChatGPT desktop, Agents SDK, Responses API)
- Apr 2025: Google DeepMind confirms Gemini support
- 2025: Microsoft partners for C# SDK
- Community: Thousands of MCP servers built

**Security Issues** (Apr-Jul 2025):
- Prompt injection vulnerabilities
- Tool permission combinations enabling file exfiltration
- Lookalike tools silently replacing trusted ones
- ~2,000 exposed MCP servers lacked authentication (Knostic research)

**Relevance to Chapter 07**:
- MCP is de facto standard for agent-tool connections
- Security concerns MUST be discussed for legal AI
- C# SDK relevant for enterprise .NET environments

### A2A (Agent-to-Agent Protocol)

**Key Features**:
- JSON-RPC 2.0 over HTTP(S), SSE
- Agent Card for capability advertisement
- Long-running tasks support
- Enterprise authentication
- Nov 2025: gRPC support added

**Distinction from MCP**:
- MCP: Agent-to-tool communication
- A2A: Agent-to-agent communication
- Complementary, not competing

**Relevance to Chapter 07**:
- Critical for multi-agent systems
- Shows agents need BOTH protocols

### Emerging Protocol Landscape

Per IBM overview, emerging standards include:
- MCP (Anthropic) — agent-to-tool
- A2A (Google/Linux Foundation) — agent-to-agent
- ACP (IBM/Linux Foundation) — agent communication
- OASF (Open Agentic Schema Framework)
- ANP (Agent Network Protocol)
- AG-UI (Agent-User Interaction)
- Agent Rules (AGENTS.md standard)

### Vulnerabilities and Incidents

**CVE-2025-32711** (Microsoft 365 Copilot):
- CVSS 9.3 (high severity)
- AI command injection enabling data theft
- Shows real attack vectors against production agents

**MCP Security Issues**:
- ~2,000 exposed servers without authentication
- Prompt injection, file exfiltration risks

### Security Frameworks

**AWS Agentic AI Security Scoping Matrix**:
- Four architectures based on connectivity/autonomy
- Key principle: graceful degradation
- Detective controls can inject restrictions or disable agents

**Microsoft Agentic Zero Trust**:
- Assume breach
- Never implicitly trust
- Explicit verification for humans/devices/agents
- Limit access to minimum needed
