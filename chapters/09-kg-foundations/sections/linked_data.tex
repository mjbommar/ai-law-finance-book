\section{Linked Data Principles}
\label{sec:kg-foundations-linkeddata}

Linked Data rests on four simple practices: use IRIs as names for things, use HTTP IRIs so people and machines can look them up, provide useful information (RDF) when IRIs are dereferenced, and include links to other IRIs to enable discovery. \autocite{tbl-linked-data}

\begin{keybox}[title={Practical Guidance for Legal/Financial Data}]
\begin{itemize}
  \item Prefer persistent HTTP IRIs for entities (courts, statutes, issuers, instruments).
  \item Publish minimal RDF descriptions and link to authoritative registries.
  \item Reuse standard vocabularies; only mint new terms when necessary.
\end{itemize}
\end{keybox}
